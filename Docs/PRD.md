# PRD: 神戸電子 OB・OG ポータル (仮)

## 1. プロジェクトの背景と目的

**背景**

神戸電子専門学校において、卒業生と在校生の接点が属人的（先生経由など）になっており、知見が蓄積されていない。

**目的**

OB・OGの就職実績や経験を可視化し、在校生がキャリア形成のロールモデルを見つけ、直接コンタクトを取れる文化を醸成する。

## 2. ターゲットユーザー

**OB・OG**

- 自身の経験を後輩に伝えたい
- 母校に貢献したい卒業生

**在校生**

- 就職活動の参考にする
- 特定の企業に内定した先輩の話を聞きたい学生

**管理者**

- データの健全性を管理する教職員
- 将来的な拡張機能

## 3. ユーザー体験 (User Stories)

**OB・OGとして**

- 自分の内定先や連絡先を登録できたい
- 後輩からの相談を受け付けられるようにしたい

**在校生として**

- 志望企業や同じ学科の先輩を検索したい
- その人のアドバイスや連絡先を確認したい

## 4. 機能要件 (MVP範囲)

### 4.1 OB・OG 一覧表示機能

**内容**

- 登録済みユーザーのカード型リスト表示

**表示項目**

- 名前（ニックネーム可）
- 卒業年度
- 学科
- 内定先・勤務先

**フィルタ/検索**

- 学科別絞り込み
- キーワード検索（企業名など）

### 4.2 OB・OG プロフィール詳細・登録機能

**登録項目**

- 卒業年度
- 学科
- 内定先
- 備考（後輩へのメッセージ）
- メールアドレス

**編集機能**

- 一度登録した情報を後から修正できる

**公開制御**

- 連絡の可否の設定
- 公開/非公開の切り替え

### 4.3 認証機能

**実装**

- Google ログイン
- ログイン済みユーザーのみが一覧・詳細を閲覧可能（情報のクローズド性確保）

## 5. 非機能要件

**セキュリティ**

- ログインなしでの閲覧不可
- APIエンドポイントの保護

**パフォーマンス**

- App Router の Server Components を活用した高速なページ遷移

**レスポンシブ**

- スマートフォン（学生の利用）に対応
- PC（登録作業）に対応

## 6. 成功指標 (KPI)

- OB・OGの登録数
- 在校生による一覧ページの閲覧数（アクティブユーザー数）

## 7. 今後の拡張予定 (V2以降)

- サイト内メッセージ機能（メールアドレス非公開でのやり取り）
- 企業別口コミ・選考体験記の投稿
- 管理者用ダッシュボード（不適切ユーザーの削除等）
